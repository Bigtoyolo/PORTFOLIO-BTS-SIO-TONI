Mission 4

Création d’un projet Jira à partir d’une demande ServiceNow

1. Accéder à la création du projet

Depuis Jira, cliquez sur le menu Projects situé dans la barre supérieure, puis sélectionnez Create project. Cette action ouvre l’assistant de création qui permettra de configurer un nouveau projet selon les besoins définis dans la demande ServiceNow.

La création d’un projet doit toujours être réalisée à partir d’une demande officielle. Les demandes reçues par email ou via Teams ne doivent pas être traitées directement.

2. Sélectionner le modèle et le type de projet

Dans la liste des modèles disponibles, sélectionnez Software Development, puis choisissez le modèle Kanban. Ce modèle est celui qui doit être utilisé par défaut, sauf indication contraire dans la demande.

Après avoir cliqué sur Use Template, Jira demande de choisir le type de projet. Il est obligatoire de sélectionner Company-managed Project afin de garantir une gestion centralisée des permissions, des workflows et des configurations.

Le type Team-managed Project ne doit jamais être utilisé.

3. Renseigner les informations du projet

Saisissez le nom du projet exactement comme indiqué dans la demande ServiceNow. Vérifiez attentivement l’orthographe, les majuscules, les acronymes et la convention de nommage utilisée.

L’option Share settings with an existing project ne doit être cochée que si cela est explicitement demandé dans le ticket. Dans tous les autres cas, laissez cette option désactivée.

Cette étape est essentielle, car elle définit l’identité du projet et sa configuration initiale.

4. Ajouter les membres de l’équipe

Ajoutez les utilisateurs mentionnés dans la demande ServiceNow. Avant l’ajout, vérifiez que chaque utilisateur possède déjà un compte Jira.

Si un utilisateur n’existe pas dans Jira, contactez le demandeur afin qu’un ticket de création de compte soit ouvert. Ne créez pas de compte utilisateur dans le cadre de la création du projet.

Si aucun membre n’est indiqué dans la demande, cliquez simplement sur Skip pour passer à l’étape suivante.

5. Modifier le Project Lead

Une fois le projet créé, Jira attribue automatiquement le rôle de Project Lead à la personne ayant créé le projet. Cette configuration doit être modifiée.

Accédez aux Project Settings puis remplacez le Project Lead par la personne responsable du projet. Dans la plupart des cas, il s’agit soit du demandeur, soit de son manager.

Si cette information n’est pas précisée dans la demande, contactez le créateur du ticket afin d’obtenir une confirmation. Le compte du support ou de l’administrateur ayant créé le projet ne doit jamais rester défini comme Project Lead.

6. Bonnes pratiques et points de vigilance

Toujours utiliser un projet Company-managed et le modèle Kanban, sauf demande spécifique. Vérifiez systématiquement les informations présentes dans le ticket avant la création du projet et contactez le demandeur lorsque certaines données sont manquantes.

Évitez les erreurs courantes telles que la création d’un projet à partir d’une demande informelle, la sélection d’un projet Team-managed, le partage non autorisé des paramètres avec un autre projet ou encore le maintien de votre propre compte comme Project Lead.

Le respect de ces règles garantit une administration cohérente et conforme aux standards de l’entreprise.

7. Ajouter un utilisateur dans Jira

Depuis l’administration Atlassian, accédez à la section Utilisateurs puis cliquez sur Inviter des utilisateurs.

Cette fonctionnalité permet d’ajouter de nouveaux collaborateurs dans l’environnement Jira et Confluence afin qu’ils puissent accéder aux applications nécessaires à leurs activités.

Avant toute création, vérifiez que la demande a été validée et que les accès demandés sont clairement définis.

8. Renseigner les informations de l’utilisateur

Dans la fenêtre d’invitation, saisissez l’adresse e-mail professionnelle de l’utilisateur dans le champ prévu à cet effet.

Plusieurs adresses peuvent être ajoutées simultanément si plusieurs utilisateurs doivent recevoir les mêmes accès. Sélectionnez ensuite les applications auxquelles l’utilisateur doit avoir accès. Dans la majorité des cas, les accès sont définis dans la demande et concernent principalement Jira et Confluence.

Attribuez uniquement les applications nécessaires afin de respecter le principe du moindre privilège.

9. Associer l’utilisateur à un groupe

Sous les applications se trouve le champ Abonnement du groupe. Cette étape est essentielle car elle permet d’associer l’utilisateur à un ou plusieurs groupes de sécurité définissant ses droits d’accès aux projets et espaces de travail.

Les groupes disponibles sont référencés dans un fichier Excel de suivi contenant le nom du groupe, le périmètre d’accès associé, les responsables du groupe et les projets concernés.

Sélectionnez le groupe correspondant aux besoins exprimés dans la demande. En cas de doute sur le groupe à utiliser, consultez le fichier de référence ou contactez le demandeur avant de poursuivre.

10. Définir le rôle et les permissions de l’utilisateur

Après avoir sélectionné le groupe, vérifiez le rôle attribué à l’utilisateur. Par défaut, le rôle User doit être utilisé dans la majorité des cas. Ce rôle permet d’accéder aux applications sans disposer de privilèges administratifs.

Les autres rôles doivent être attribués uniquement lorsqu’ils sont explicitement demandés et validés : User pour un accès standard aux applications, App Admin pour l’administration de l’application concernée, Guest pour un accès limité à des espaces ou projets spécifiques, et User Access Admin pour la gestion des utilisateurs et des groupes.

L’attribution d’un rôle administratif doit toujours être justifiée et approuvée.

11. Ajouter l’utilisateur dans son équipe Tempo

Une fois l’utilisateur créé dans Jira, accédez à Tempo Teams. Les équipes Tempo permettent de gérer l’affectation des ressources, le suivi de la capacité des équipes et les droits liés aux projets.

Recherchez l’équipe du manager concerné en saisissant son nom dans la barre de recherche. Sélectionnez ensuite l’équipe correspondante afin d’afficher sa liste de membres.

12. Ajouter le collaborateur dans l’équipe du manager

Depuis l’équipe Tempo du manager, cliquez sur Ajouter un membre. Dans la fenêtre qui s’ouvre, recherchez l’utilisateur précédemment créé, sélectionnez son nom, conservez le rôle Membre sauf demande spécifique, puis définissez l’engagement à 100 % par défaut.

Complétez les dates d’entrée ou de sortie si elles sont indiquées dans la demande. Vérifiez également les éventuelles compétences associées si elles sont utilisées par l’équipe.

Cliquez ensuite sur Sauvegarder pour finaliser l’ajout. L’utilisateur devient alors membre de l’équipe du manager et peut être affecté aux projets gérés par cette équipe.

13. Vérifications finales

Avant de clôturer la demande, vérifiez que l’utilisateur apparaît bien dans Jira, que les applications demandées ont été attribuées, que le bon groupe de sécurité a été sélectionné et que les permissions correspondent à la demande.

Vérifiez également que l’utilisateur est présent dans l’équipe Tempo du manager et qu’aucun droit administratif n’a été attribué sans validation.

Une fois ces contrôles effectués, vous pouvez confirmer la création des accès et clôturer la demande.
